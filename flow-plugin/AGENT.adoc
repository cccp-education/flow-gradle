= AGENT.adoc — Flow Gradle

_Produit_ : Plugin Gradle d'orchestration — merge, auto-close, conventional commits, CI triggers
_Licence_ : Apache 2.0 (OSS)
_DAG_ : N4 — Plugin Gradle de gouvernance (consomme ticket + review)
_Dépendance_ : ticket-gradle (compileOnly, lecture tickets), review-gradle (compileOnly, lecture review), GitHub API

== Principe

Le plugin `flow` orchestre le **workflow de merge** : quand la review est terminée
et les quality gates passées, il déclenche le merge, l'auto-close des tickets liés,
et le déclenchement du pipeline CI.

Pipeline :
  Ticket (Harlem) + Review (Baltimore) ──→ merge PR ──→ auto-close ──→ CI trigger

Cas d'usage :
- Merge automatique quand review IA + humaine OK
- Auto-close des GitHub Issues liées (Fixes #XX dans le commit)
- Validation conventional commits avant merge
- CI trigger : runner-gradle assembleCompositeContext après merge
- Rapport de merge (quoi mergé, quels tickets fermés)

== Règles Absolues

=== 1. COMMITS/GIT — INTERDICTION FORMELLE sans permission pour merge/push
=== 2. PUBLISH — INTERDICTION FORMELLE sans permission
=== 3. DAG N4 — consomme N4 (ticket-gradle, review-gradle), jamais importé par N3/N2
=== 4. PROCEDURE DE FIN DE SESSION — 6 ETAPES OBLIGATOIRES
=== 5. HOOK OUVERTURE — Checklist SESSION_CHECKLIST.adoc obligatoire
=== 6. ZERO secret/token/api key en dur (⚠️ GITHUB_TOKEN en env var uniquement)
=== 7. MERGE REQUIERT REVIEW HUMAINE — le flow vérifie l'approval PR avant merge

== Tâches Principales

  ./gradlew flowStatus             → statut du flow (tickets → PRs → review → ready?)
  ./gradlew flowMerge              → merge PR si toutes les gates passent
  ./gradlew flowClose              → auto-close les tickets liés après merge
  ./gradlew flowCICD               → déclencher pipeline CI après merge
  ./gradlew flowReport             → rapport de merge (AsciiDoc)
  ./gradlew check                  → tests

== Stack

- Kotlin — DSL Gradle
- GitHub API (gh CLI wrapper)
- ticket-gradle — dépendance compileOnly, lecture tickets
- review-gradle — dépendance compileOnly, lecture statut review
- runner-gradle — CI trigger post-merge

== Quality Gates (avant merge)

[cols="2,3"]
|===
| Gate | Vérification
| Review IA | Score de qualité review-gradle ≥ seuil configuré
| Review humaine | PR approved par au moins 1 reviewer
| CI | Tests passent (workflow GitHub Actions OK)
| Conventional commits | Format vérifié (feat:/fix:/chore:)
| Secrets | Aucun token/password dans le diff
| DAG | Pas d'import N3 → N2 (violation contrat)
|===

== Ne remplace PAS opencode

Le plugin orchestre le workflow, mais le merge reste une décision humaine.
Le flow automatisable = ce qui est déterministe (format, secrets, CI).
Le flow humain = approval PR, décision architecturale.
